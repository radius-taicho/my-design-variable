# my-design-variable

> 定数を集め、変数を育て、デザインの応用力を手に入れる。

```
日常の気づき・実務の経験・理論の学び
            ↓
      hypothesis-variables（仮説の変数）
            ↓
      実務で試す・理論と照合する
            ↓
        my-variables（自分の変数として応用できる状態）
```

## ノートの書き方
- すべてのノートに **「自分メモ」** セクションを入れる
- `[[ノート名]]` でカテゴリをまたいで繋げる（例: `[[01_Theory/Typography/rounded-typeface-psychology]]`）
- frontmatterのtagsで横断検索できるようにする

## 作業フロー（AIとの共同作業）

このナレッジベースはAI（Claude）と一緒に育てることを想定している。

### 仮説変数を追加したいとき
```
1. 思いついたことを殴り書きで伝える
2. ClaudeがHypothesis-variablesに整理して配置する
3. 新しいカテゴリが必要な場合はClaudeが作成する
```

### 理論ノートを作りたいとき
```
1. 「〇〇について調べてまとめて」と伝える
2. ClaudeがTheoryの適切なフォルダにノートを作成する
3. 自分メモ欄だけ自分で埋める
```

### 案件の振り返りをしたいとき
```
1. 案件の概要と気づきを話す
2. ClaudeがPractice/Projectsにproject-templateで整理して作成する
```

## 今後のカスタマイズ予定（Obsidian）

### 優先度高
- **Templaterプラグイン** — テンプレートに日付や入力欄を自動挿入できる。Templates/のノートをより便利に使えるようになる
- **Graph View の活用** — `[[リンク]]` が増えてくるとノート間の繋がりが視覚的に見えてくる。_Variablesと各Theoryノートの繋がりを確認するのに使えそう

### 優先度中
- **Dataviewプラグイン** — ノートをデータベースのように扱える。例えば「タグがtheory/typographyのノートを一覧表示する」といったことが可能になる
- **Canvasの活用** — 無限ボードにノートや画像を自由に配置できる。デザイン分析やブレストの可視化に使えそう
- **Kanbanプラグイン** — 「調べたいタスク」の管理に使えそう

### 将来的に
- **Dataview + タグ管理** — 仮説変数が増えてきたら、タグで「検証済み/未検証/棄却」などを管理できるようにする

## 今後のカスタマイズ予定（情報収集の自動化）

確度の高い記事・情報を自動でナレッジベースに取り込む仕組みを作る。
対象は以下の3ジャンル。

- **Visual Design** — UI/UX・タイポグラフィ・配色・レイアウトなど
- **Frontend & Implementation** — HTML/CSS・パフォーマンス・アクセシビリティなど
- **Industry & Career** — 業界動向・採用・ポートフォリオ・ツール変遷など

### 優先度高
- **Omnivore（無料）+ ObsidianプラグインでRSS/記事収集** — ブラウザ拡張で保存した記事をコマンド一つでObsidianに取り込む。`05_References`や`06_Tech`への自動振り分けも可能

### 優先度中
- **GitHub Actions で定期収集** — Hacker News・Dev.to・Zennなどから指定キーワードでフィルタリングして毎朝自動でmdファイルを生成・pushする
- **RSSソースの選定** — 信頼度の高いソースをリスト化する（例: CSS-Tricks・Smashing Magazine・web.dev・Zenn・Dev.to）

### 将来的に
- **AIによる要約 + 自分メモ欄の自動生成** — 取り込んだ記事をClaudeが要約し、仮説変数の候補まで提示してくれる仕組み

## 変数の育て方

理論を学んで → 実務で試して → 気づいたことを書く。
それを10年続ければ、ここが自分だけのデザイン哲学になる。
